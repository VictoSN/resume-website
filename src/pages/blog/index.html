Linux Mail Server

By: Victorio Suwita Nanda
23 November 2024

1.    Introduction

E-mail remains an important communication tool in the digital age, as they are secure and effective. As a Linux Network Administration, the task of setting up a fully functional e-mail server is crucial to deploy a seamless network environment. This paper outlines the steps required in order to achieve the requested task, beginning with establishing a static IP address and configuring DNS and DHCP.

The focus point of the paper is on installing and configuring Postfix as the Mail Transfer Agent (MTA) and Dovecot as the IMAP/POP3 server, allowing users to send and receive e-mails through third-party clients such as Thunderbird. The Encryption will be provided by the SSL/TLS to ensure secure e-mail transmission and retrieval. This essay also delves into some of the troubleshooting involved in setting up the e-mail server.

5.    Chapter 4: SSL

SSL (Secure Sockets Layer) is used to encrypt the contents of the e-mail communication. Certificates and keys will be used for authentication. This chapter will guide you through the installing, configuring, and testing SSL for Postfix.

5.1.          Installation

For this task, OpenSSL will be used to generate SSL certificates and keys. OpenSSL is a widely used tool for encryption and security protocols.

Figure 34

Figure 34: To install OpenSSL, run the command 'sudo dnf install openssl'.

5.2.          Configuration

The configuration for OpenSSL involves generating SSL certificates and keys, and configuring them to work with Postfix.

 

Create the a directory to store the certificate and key files for Postfix.

Figure 35

Figure 35: Two directories are required one for storing the certificate and the other for the key, use the command 'sudo mkdir -p /etc/ssl/certs /etc/ssl/keys' to create both.

Figure 36

Figure 36: To generate the certificate and key file, run the command:

sudo openssl req -x509 -nodes -keyout /etc/ssl/keys/postfix.key -out /etc/ssl/certs/postfix.crt -days 365 -newkey rsa:2048

 

This command will ask you to enter your information such as country, state and organization.

Figure 37

Figure 37: Enter the appropriate information for your network system.

 

After SSL is generated, we need to go back to Postfix's main configuration file.

Figure 38

Figure 38: Like before use 'sudo nano /etc/postfix/main.cf' to open the main configuration of Postfix.

Figure 39

Figure 39: Set the certificate and key to their location in the Rocky file.

Figure 40

Figure 40: Ensure the correct configuration to allow for SSL and Postfix to work seamlessly.

 

Restart Postfix to apply the changes made.

Figure 41

Figure 41: Execute 'sudo systemctl restart postfix' to restart Postfix.

Figure 43

Figure 43: In the Ubuntu client use 'sudo apt install thunderbird -y' to check the Linux repository and install Thunderbird.

6.2.          Configuration

To ensure proper communication between the server and client, both systems must be configured.

 

For some, the firewall may block the ports required to use the e-mail services. We only need the ports for 587 and 993, but other ports can be used to as alternative.

Figure 44

Figure 44: Use the commands 'sudo firewall-cmd 'add-port=25/tcp 'permanent' to add an exception on Rocky server. Repeat for '465', '587' and '993' and reload the firewall using 'sudo firewall-cmd 'reload'.

 

The same steps apply to the Ubuntu client.

Figure 45

Figure 45: For the Ubuntu client, use 'sudo ufw allow 25/tcp'. Repeat for '465', '587' and '993' and reload the firewall using 'sudo ufw reload'.

8.    Conclusion

This project provides a comprehensive walkthrough of setting up a functional email server using Postfix as the Mail Transfer Agent (MTA), Dovecot as the IMAP/POP3 server, and Thunderbird as the email client. The configuration includes the usage of essential components such as DNS and DHCP services to ensure smooth communication and email delivery. Security measures, including the usage of OpenSSL's SSL/TLS encryption, were used to safeguard email transmissions and user data.

Beyond the steps of installation, configuration and testing, this project also includes troubleshooting to provide a deeper understanding of managing and resolving potential issues. This setup have the potential to be scaled up for an enterprise-level e-mail system. Overall, this project highlights the proper configuration for an e-mail system.
